## Overview

### Problem
The app is functionally complete but has several accessibility (a11y) gaps and minor responsive design issues that should be addressed before launch:
- Icon-only buttons have `title` attributes but not `aria-label` — screen readers ignore `title`
- Dynamic status messages update silently — screen readers won't announce them
- The timer SVG has no semantic role
- The admin quiz table has 7 columns that overflow on narrow viewports
- Metadata description is a placeholder

### Solution
Targeted fixes across ~7 files. No new npm dependencies. The live game screens (LobbyView, QuestionView) are intentionally projector-optimized — no responsive changes needed there.

### Goals
- [ ] All interactive icon-only buttons have `aria-label`
- [ ] Dynamic status messages use `aria-live` / `role="status"`
- [ ] Timer has `role="timer"` for screen readers
- [ ] Admin quiz table usable at 375px viewport width
- [ ] Metadata description updated
- [ ] `yarn lint` clean, `yarn test` 368 tests pass

## Implementation Steps

### Step 1: aria-label on icon-only buttons
**File:** `src/components/admin/quiz-list.tsx`

All action buttons show only an icon. The `title` attribute is not read by screen readers.

Add `aria-label` to each button:
- Edit button (line ~119): `aria-label="Edit quiz"`
- Open Lobby button (line ~128): `aria-label="Open lobby"`
- Open Live View button (line ~138): `aria-label="Open live view"`
- Delete button (line ~148): `aria-label="Delete quiz"`

Keep `title` as well (tooltip on hover for sighted users).

---

### Step 2: Better alt text for question media
**File:** `src/components/host/live/question-view.tsx`

Current: `alt="Question media"` (line ~69) — generic, fails WCAG SC 1.1.1.

Fix: `alt={activeQuestion.text}` — describes the content the image accompanies.

---

### Step 3: aria-live for dynamic status messages
**File:** `src/components/player/player-session-screen.tsx`

Two elements update dynamically but are not announced to screen readers:

**reconnectedToast** (line ~237):
```tsx
<div
  role="status"
  aria-live="polite"
  className="rounded-lg border border-primary/40 bg-primary/10 p-3 text-sm text-primary"
>
```

**submissionMessage** (line ~?? — wherever the submissionMessage is rendered):
```tsx
<p role="status" aria-live="polite" className="...existing classes...">
```

**error** div (the fetch error display):
```tsx
<div role="alert" aria-live="assertive" className="...existing classes...">
```

---

### Step 4: role="timer" on countdown
**File:** `src/components/host/timer-countdown.tsx`

The outer `<div className="flex flex-col items-center gap-4">` (line 101) wraps the timer but has no semantic role.

Add `role="timer"` and `aria-label` to communicate the time remaining:
```tsx
<div
  className="flex flex-col items-center gap-4"
  role="timer"
  aria-label={`${formatTime(displayTime)} ${showElapsed ? 'elapsed' : 'remaining'}`}
>
```

### Step 6: Responsive admin quiz table
**File:** `src/components/admin/quiz-list.tsx`

At 375px the 7-column table overflows. Admin is primarily desktop but should degrade gracefully.

Hide the three least-critical columns on mobile (keep Title, Status, Join Code, Actions):

**TableHead cells** — add `hidden sm:table-cell`:
```tsx
<TableHead className="hidden sm:table-cell">Questions</TableHead>
<TableHead className="hidden sm:table-cell">Players</TableHead>
<TableHead className="hidden sm:table-cell">Time/Question</TableHead>
```

**TableCell cells** (inside the `.map()`) — add `hidden sm:table-cell`:
```tsx
<TableCell className="hidden sm:table-cell">{quiz.questionCount}</TableCell>
<TableCell className="hidden sm:table-cell">{quiz.playerCount}</TableCell>
<TableCell className="hidden sm:table-cell">{quiz.timePerQuestion}s</TableCell>
```

### Step 7: Admin header email truncation
**File:** `src/app/(admin)/admin/layout.tsx`

The `<span>` showing user email can cause the header to wrap at small viewports.

Add `hidden md:inline` to hide the email below `md` breakpoint (768px):
```tsx
{userEmail && (
  <span className="hidden md:inline text-sm text-muted-foreground">{userEmail}</span>
)}
```

## Technical Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| No axe-core install | Manual targeted fixes | Avoids new dependency; specific issues already known |
| `aria-label` not `title` | Add both | `title` is for hover tooltips; `aria-label` is for screen readers |
| Hide columns vs card layout | `hidden sm:table-cell` | Minimal change; admin is desktop-primary |
| Implicit `<label>` wrapping | Keep as-is | player-join-form.tsx already uses correct wrapping pattern |
| Live screens responsive | No changes | Projector-optimized; intentionally large text |
